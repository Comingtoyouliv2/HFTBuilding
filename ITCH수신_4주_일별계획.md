# ITCH 수신기 — 4주 일별 학습 계획

> 전제: **하루 2~3시간**, 기초 C++ 문법 보유, 개발 Mac + 저수준 실습 Linux x86 컨테이너.
> 목표: 4주 안에 Stage 1~10을 **"통과"(작동하는 코드 + 효과 측정)** 수준까지.

**사용법**
- 하루 = 작은 산출물 하나. 코드는 항상 직접 타이핑(복붙 X).
- 각 날 **체크포인트**를 통과해야 다음으로. 못 하면 그날 더 보거나 주말 버퍼 날로 미룸.
- 주 7일 중 **6일 학습 + 1일 복습/버퍼**. 막히면 죄책감 없이 버퍼 날 사용.
- "막힘 30분 규칙": 30분 막히면 메모해두고 질문하거나 다음 날로.

|-----|------|----------------|---------|--------------|

## 주별 한 줄 요약

- **Week 1**: 바이트 → 메시지 → 파일. zero-copy 파싱의 기초 (가장 할 만함)
- **Week 2**: UDP 수신 + 올바른 측정 + 캐시 레이아웃 진입
- **Week 3**: prefetch + 동시성(mutex→lock-free). **제일 어려움, 여유 필수**
- **Week 4**: affinity + kernel bypass + 전체 통합

## 현실적 조언
- **Week 3에서 막히는 게 정상이에요.** lock-free·메모리 순서는 원래 어려워요. 완벽 이해보다 "동작시키고 효과 측정"이 목표.
- 매주 **측정 리포트 1장**씩 남기세요. 나중에 FPGA RTL과 비교할 소중한 기준이 돼요.
- 일정이 밀려도 OK. 버퍼 날이 그래서 있어요. **중요한 건 매일 코드를 만지는 습관.**
